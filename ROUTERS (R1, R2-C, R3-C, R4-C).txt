ROUTERS (R1, R2-C, R3-C, R4-C)
enable

Entra al modo privilegiado para poder configurar

configure terminal

Entra al modo de configuración global

hostname R1

Le pone nombre al router

ip domain-name itla.edu.do

Define el dominio, necesario para generar las llaves SSH

crypto key generate rsa modulus 1024

Genera las llaves de encriptación RSA para SSH

ip ssh version 2

Activa SSH versión 2, más seguro que la versión 1

username 20252207 secret Fe2024
username opelegrino secret Fe2024

Crea los usuarios con contraseña encriptada para acceder por SSH

line vty 0 4
 login local
 transport input ssh

Configura las líneas virtuales (acceso remoto) para que solo acepten SSH con usuarios locales

 R1 – Específicos

interface GigabitEthernet0/0
 no shutdown

Enciende la interfaz física que va hacia SW1, necesaria para que las subinterfaces funcionen

interface GigabitEthernet0/0.300
 encapsulation dot1Q 300

Crea la subinterfaz para VLAN 300 y la etiqueta con dot1Q para que el switch pueda identificarla

ip address 192.168.30.2 255.255.255.0

Le asigna la IP a R1 dentro de la VLAN 300

ip helper-address 192.168.32.1

Redirige las solicitudes DHCP de los clientes hacia R3-C que es el servidor DHCP

standby 1 ip 192.168.30.1

Define la IP virtual compartida entre R1 y R4-C que usan los clientes como gateway

standby 1 priority 90

Le da prioridad 90 a R1, como es menor que R4-C (110) queda como Standby (respaldo)

standby 1 preempt

Permite que R1 tome el control si R4-C falla y luego vuelve

interface GigabitEthernet0/1
 ip address 192.168.100.1 255.255.255.252

Interfaz que conecta R1 con R2-C, red /30 porque solo necesita 2 hosts

ip route 192.168.32.0 255.255.255.0 192.168.100.2

Ruta estática para llegar a la LAN del servidor, pasando por R2-C

ip route 192.168.100.4 255.255.255.252 192.168.100.2
ip route 192.168.100.8 255.255.255.252 192.168.100.2

Rutas para llegar a las otras conexiones seriales de la topología pasando por R2-C

R3-C – Específicos
ip dhcp excluded-address 192.168.30.1 192.168.30.10

Reserva las primeras 10 IPs de VLAN 300 para no asignarlas por DHCP (gateways, switches, etc)

ip dhcp pool VLAN300
 network 192.168.30.0 255.255.255.0
 default-router 192.168.30.1
 dns-server 192.168.32.5
 lease 1

R4-C – Específicos
interface GigabitEthernet0/1.300
 encapsulation dot1Q 300

Crea la subinterfaz para VLAN 300 hacia SW2

standby 1 priority 110

Le da prioridad 110 a R4-C, como es mayor que R1 (90) queda como Router ACTIVO para HSRP

standby 1 preempt

Permite que R4-C retome el rol de Activo automáticamente si vuelve después de una falla

interface Serial0/0/0
 ip address 192.168.100.9 255.255.255.252

Interfaz serial que conecta R4-C con R2-C

ip route 192.168.32.0 255.255.255.0 192.168.100.10
ip route 192.168.100.0 255.255.255.252 192.168.100.10
ip route 192.168.100.4 255.255.255.252 192.168.100.10

 SWITCHES (SW1 y SW2)
vlan 300
 name Estudiantes

Crea la VLAN 300 y le da el nombre Estudiantes para identificarla

spanning-tree vlan 300 priority 4096

Le da a SW1 la prioridad más baja posible para que sea elegido Puente Raíz del STP, controlando el flujo de tráfico y evitando bucles

interface GigabitEthernet0/2
 switchport mode trunk
 switchport trunk native vlan 999
 switchport trunk allowed vlan 300,500,999

Configura el puerto hacia el router como trunk para que pueda transportar todas las VLANs, la nativa es 999

interface range FastEthernet0/23 - 24
 channel-group 1 mode active

Agrupa los puertos 23 y 24 en un EtherChannel con protocolo LACP (active), aumenta el ancho de banda y da redundancia entre switches

interface FastEthernet0/9
 switchport mode access
 switchport access vlan 500

Pone el puerto 9 en modo acceso para que solo transporte tráfico de VLAN 500 donde está PC Duarte

interface range FastEthernet0/1 - 8
 switchport mode access
 switchport access vlan 999
 shutdown

Los puertos no usados se ponen en VLAN 999 y se apagan por seguridad para que nadie los use sin autorización

interface vlan 999
 ip address 192.168.99.10 255.255.255.0

Le da una IP al switch para poder administrarlo remotamente por SSH desde cualquier red

ip default-gateway 192.168.99.1

Define el gateway del switch para que pueda comunicarse fuera de su red local, apunta al gateway virtual HSRP

end
write memory

Sale del modo configuración y guarda todo en la memoria permanente para que no se pierda al reiniciar
